  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),

  created_at TIMESTAMPTZ DEFAULT now(),

CREATE TABLE entry_sections (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  entry_id UUID REFERENCES journal_entries(id) ON DELETE CASCADE,
  user_id UUID REFERENCES users(id),
  content TEXT,
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now(),

);

## CRITICAL TECHNICAL RULES

1. **FK Hints on ALL Supabase nested queries.** Every .select() joining tables MUST use !foreign_key syntax:
   .select('*, sections:entry_sections!entry_id(*), photos:entry_photos!entry_id(*)')
   Without hints, PostgREST returns 400. This is the #1 bug source.

2. **Optimistic UI everywhere.** Update cache immediately, sync in background. Show "Saving..." / "Saved".

3. **Mobile-first.** 375px first. 44px tap targets. Bottom nav with safe area padding.

4. **Photo resize:** 1600px max, JPEG 85%, canvas-based, Math.round() dimensions, revoke object URLs.

5. **Auto-save:** 1.5s debounce on journal editing.

6. **PWA: skipWaiting + clientsClaim** for immediate updates.
