This document contains everything needed to rebuild The Kristory

# ============================================================
# 1. WHAT IS THE KRISTORY?
# ============================================================

The Kristory is a shared daily journal PWA (Progressive Web App)
for two users — Chris and Krista. It's a digital scrapbook and
daily diary where they document their life together through
narrative entries, photos, tagged items (movies, restaurants, etc.),
trips, recipes, and memories.

It also includes "The Babory" — a family-facing baby feed at
/family where family members can follow baby updates, view shared
photos, and suggest baby names via a PIN-protected link.

URL: https://the-kristory.vercel.app
Family feed: https://the-kristory.vercel.app/family

Frontend:

  - Deploy command: cd /d/AI/Claude/Kristory; npx vercel --prod
  - Domain: the-kristory.vercel.app

Git:
  - GitHub repo under homer31383
  - Push manually: cd D:\AI\Claude\Kristory && git push

Environment Variables (in .env):
  - VITE_SUPABASE_URL — your Supabase project URL
  - VITE_SUPABASE_ANON_KEY — your Supabase anon/public key

The app shares a Supabase instance with other apps (Axiom Tasks,
Stint, Pulse, Stint Ledger). All Kristory tables are standalone
with unique names.

# 4. COMPLETE DATABASE SCHEMA
# ============================================================

ALL tables have RLS (Row Level Security) DISABLED.
The app is for 2 trusted users — no auth, no RLS.

CRITICAL: All Supabase queries with nested selects MUST use
explicit FK hints (e.g. !entry_id, !category_id) or PostgREST
returns 400 errors.

--- SHARED (pre-existing) ---

TABLE: users
  id          UUID PK
  name        TEXT NOT NULL
  created_at  TIMESTAMPTZ DEFAULT now()

  Data:
    Chris  — 9df51388-65e8-44bf-bfad-65729744190b
    Krista — cab9d09c-0083-4ce1-aaf5-9a045d8a4315

--- JOURNAL ---

TABLE: journal_entries
  id          UUID PK DEFAULT gen_random_uuid()
  entry_date  DATE UNIQUE NOT NULL
  created_at  TIMESTAMPTZ DEFAULT now()
  updated_at  TIMESTAMPTZ DEFAULT now()

TABLE: entry_sections
  id          UUID PK DEFAULT gen_random_uuid()
  entry_id    UUID FK → journal_entries.id ON DELETE CASCADE
  user_id     UUID FK → users.id
  content     TEXT (stored as sanitized HTML)
  created_at  TIMESTAMPTZ DEFAULT now()
  updated_at  TIMESTAMPTZ DEFAULT now()
  fts         TSVECTOR GENERATED ALWAYS AS (to_tsvector('english', coalesce(content, ''))) STORED
  UNIQUE (entry_id, user_id)

  INDEX: idx_entry_sections_fts ON entry_sections USING GIN (fts)

TABLE: entry_photos
  id            UUID PK DEFAULT gen_random_uuid()
  entry_id      UUID FK → journal_entries.id ON DELETE CASCADE
  user_id       UUID FK → users.id
  storage_path  TEXT NOT NULL (format: {entry_date}/{uuid}.jpg)
  display_order INTEGER DEFAULT 0
  created_at    TIMESTAMPTZ DEFAULT now()

--- CATEGORIES & TAGGED ITEMS ---

TABLE: categories
  id          UUID PK DEFAULT gen_random_uuid()
  name        TEXT NOT NULL
  emoji       TEXT
  is_default  BOOLEAN DEFAULT false
  user_id     UUID FK → users.id (required)
  created_at  TIMESTAMPTZ DEFAULT now()

  Default categories (seeded):
    🎬 Movies, 📺 TV Shows, 🍽 Restaurants, 🍳 Home Cooking,
    🎭 Activities, 📖 Books, ✈️ Trips, 🎵 Music/Concerts,
    🛒 Shopping, 👶 Baby

TABLE: tagged_items
  id                UUID PK DEFAULT gen_random_uuid()
  entry_id          UUID FK → journal_entries.id ON DELETE CASCADE (NULLABLE)
  category_id       UUID FK → categories.id
  user_id           UUID FK → users.id
  name              TEXT NOT NULL
  rating            INTEGER CHECK (rating >= 1 AND rating <= 5)
  location_name     TEXT NULLABLE
  location_lat      DECIMAL NULLABLE
  location_lng      DECIMAL NULLABLE
  location_place_id TEXT NULLABLE
  ingredients       TEXT NULLABLE (Home Cooking recipes)
  instructions      TEXT NULLABLE (Home Cooking recipes)
  item_date         DATE NULLABLE
  created_at        TIMESTAMPTZ DEFAULT now()

  INDEX: idx_tagged_items_name USING GIN (to_tsvector('english', name))

--- RECIPE TAGS ---

TABLE: recipe_tags
  id          UUID PK DEFAULT gen_random_uuid()
  name        TEXT NOT NULL UNIQUE
  emoji       TEXT
  created_at  TIMESTAMPTZ DEFAULT now()

  Seeded: Pasta, Salad, Soup, Sauce, Sandwich, Protein,
          Baking, Sweets, Pancakes, Cookies, Sides, Breakfast

TABLE: tagged_item_recipe_tags
  tagged_item_id  UUID FK → tagged_items.id ON DELETE CASCADE
  recipe_tag_id   UUID FK → recipe_tags.id ON DELETE CASCADE
  PK (tagged_item_id, recipe_tag_id)

--- PARTICIPANTS ---

TABLE: tagged_item_participants
  tagged_item_id  UUID FK → tagged_items.id ON DELETE CASCADE
  user_id         UUID FK → users.id ON DELETE CASCADE
  PK (tagged_item_id, user_id)

--- TRIPS ---

TABLE: trips
  id               UUID PK DEFAULT gen_random_uuid()
  title            TEXT NOT NULL
  summary          TEXT NULLABLE
  cover_photo_path TEXT NULLABLE
  start_date       DATE NOT NULL
  end_date         DATE NOT NULL
  created_at       TIMESTAMPTZ DEFAULT now()

TABLE: trip_entries
  trip_id   UUID FK → trips.id ON DELETE CASCADE
  entry_id  UUID FK → journal_entries.id ON DELETE CASCADE
  PK (trip_id, entry_id)

--- BABY ---

TABLE: baby_milestones
  id              UUID PK DEFAULT gen_random_uuid()
  entry_id        UUID FK → journal_entries.id ON DELETE SET NULL (NULLABLE)
  title           TEXT NOT NULL
  milestone_type  TEXT NOT NULL DEFAULT 'custom'
  milestone_date  DATE NOT NULL
  notes           TEXT NULLABLE
  photo_path      TEXT NULLABLE
  created_at      TIMESTAMPTZ DEFAULT now()

TABLE: baby_profile
  id            UUID PK DEFAULT gen_random_uuid()
  name          TEXT DEFAULT 'Baby'
  due_date      DATE (2026-10-02)
  birth_date    DATE NULLABLE
  birth_weight  TEXT NULLABLE
  birth_length  TEXT NULLABLE
  notes         TEXT NULLABLE
  family_pin    TEXT DEFAULT '2026'
  created_at    TIMESTAMPTZ DEFAULT now()
  updated_at    TIMESTAMPTZ DEFAULT now()

TABLE: baby_name_suggestions
  id            UUID PK DEFAULT gen_random_uuid()
  name          TEXT NOT NULL
  suggested_by  TEXT NULLABLE
  created_at    TIMESTAMPTZ DEFAULT now()

--- FAMILY FEED ---

TABLE: family_posts
  id            UUID PK DEFAULT gen_random_uuid()
  entry_id      UUID FK → journal_entries.id ON DELETE CASCADE
  caption       TEXT
  published_at  TIMESTAMPTZ (should reflect journal entry date)
  user_id       UUID FK → users.id
  created_at    TIMESTAMPTZ DEFAULT now()
  updated_at    TIMESTAMPTZ DEFAULT now()

TABLE: family_post_photos
  id              UUID PK DEFAULT gen_random_uuid()
  family_post_id  UUID FK → family_posts.id ON DELETE CASCADE
  entry_photo_id  UUID FK → entry_photos.id ON DELETE CASCADE
  display_order   INTEGER DEFAULT 0

--- APP SETTINGS ---

TABLE: app_settings
  key         TEXT PK
  value       TEXT NOT NULL
  updated_at  TIMESTAMPTZ DEFAULT now()

  Known keys: 'app_pin', 'last_backup_date', 'backup_reminder'

--- SUPABASE STORAGE ---

Bucket: kristory-photos
  - Public access enabled
  - Policies: INSERT/SELECT/UPDATE/DELETE for bucket_id = 'kristory-photos'
  - Photos at: {entry_date}/{uuid}.jpg
  - Resize: 1600px max, JPEG 85%

# ============================================================
# 9. KNOWN QUIRKS
# ============================================================

- categories requires user_id (not nullable)
- tagged_items.entry_id is NULLABLE (standalone items)
- Storage bucket needs explicit RLS policies
- PWA caches aggressively — use Check for Updates
- Photo upload: Math.round() canvas dims, revoke object URLs
- Photo delete: edit mode toggle prevents fat-finger
- Family feed published_at should use entry date not share date
- No www. on Vercel URLs (SSL error)
- PowerShell: use ; not && for chaining commands
